# 新 API 后端项目启动清单

本文档用于新 API 后端项目启动时快速检查关键决策是否已经明确。它是入口清单，详细规则分别维护在：

- [PROJECT_ARCHITECTURE.md](PROJECT_ARCHITECTURE.md)：项目架构
- [API_CODE_STYLE.md](API_CODE_STYLE.md)：API 代码规范
- [API_RESULT_GUIDE.md](API_RESULT_GUIDE.md)：统一结果对象
- [AUTOFAC_GUIDE.md](AUTOFAC_GUIDE.md)：Autofac 注册及使用
- [CLASS_LIBRARY_GUIDE.md](CLASS_LIBRARY_GUIDE.md)：类库项目设计
- [REPOSITORY_GUIDE.md](REPOSITORY_GUIDE.md)：Repository 仓储规范与使用

不包含范围：

- 前端、页面模板、JavaScript、CSS
- 认证、授权、Token、权限策略细节
- Factory 相关设计
- 某个具体项目、目录或业务模块的约定

## 一、项目目标

- [ ] 明确项目要解决的核心业务问题。
- [ ] 明确首批 API 的调用方和使用场景。
- [ ] 明确哪些能力是当前项目特有，哪些能力应抽到通用层。
- [ ] 明确首版交付范围，避免一开始把可选能力全部塞进去。
- [ ] 明确非目标范围，避免后续需求边界不断扩张。

## 二、架构决策

- [ ] 已确定分层结构：API 层、业务服务层、领域层、数据访问层、基础设施层、外部集成层。
- [ ] 已确定依赖方向，避免下层反向依赖上层。
- [ ] 已确定 Controller 到 Service 到 Repository / DbContext 的调用链。
- [ ] 已确定跨模块调用方式。
- [ ] 已确定公共能力放置位置。
- [ ] 已确定是否需要外部集成层封装第三方系统。
- [ ] 已阅读并采用 [PROJECT_ARCHITECTURE.md](PROJECT_ARCHITECTURE.md)。

## 三、项目基础配置

- [ ] 已确定目标框架和语言版本。
- [ ] 已确定 nullable 策略。
- [ ] 已确定是否启用隐式 using。
- [ ] 已确定代码分析器和格式化规则。
- [ ] 已确定配置文件结构和环境区分方式。
- [ ] 已确定密钥、连接串、证书等敏感配置的管理方式。
- [ ] 已确定启动时需要校验哪些关键配置。

## 四、目录和模块

- [ ] 已建立 API、DTO、Service、领域、数据访问、基础设施等基础目录或模块。
- [ ] 已避免按技术细节过度拆分模块。
- [ ] 已避免把无关业务放进同一个大模块。
- [ ] 已确定 DTO、实体、配置对象、结果对象的放置位置。
- [ ] 已确定本地应用服务和可复用业务服务的边界。
- [ ] 已确定类库拆分、公共 API、项目引用和依赖方向。
- [ ] 已阅读并采用 [CLASS_LIBRARY_GUIDE.md](CLASS_LIBRARY_GUIDE.md)。

## 五、统一结果对象

- [ ] 已确定 API 响应统一使用结果包装对象。
- [ ] 已确定成功字段名称。
- [ ] 已确定成功码。
- [ ] 已确定错误码分段规则。
- [ ] 已确定业务失败和 HTTP 状态码的关系。
- [ ] 已确定分页响应结构。
- [ ] 已确定 traceId / requestId 字段名称。
- [ ] 已确定参数校验错误格式。
- [ ] 已确定全局异常响应格式。
- [ ] 已阅读并采用 [API_RESULT_GUIDE.md](API_RESULT_GUIDE.md)。

## 六、API 设计

- [ ] 已确定路由命名规则。
- [ ] 已确定 HTTP 方法使用规则。
- [ ] 已确定列表、详情、创建、更新、删除、业务动作接口的写法。
- [ ] 已确定分页参数名称和默认值。
- [ ] 已确定排序、过滤、搜索参数的表达方式。
- [ ] 已确定请求 DTO 和响应 DTO 的命名规则。
- [ ] 已确定已发布接口的兼容性策略。
- [ ] 已确定是否需要接口版本化。
- [ ] 已阅读并采用 [API_CODE_STYLE.md](API_CODE_STYLE.md)。

## 七、依赖注入

- [ ] 已确定依赖注入容器。
- [ ] 已确定注册入口。
- [ ] 已确定是否使用模块化注册。
- [ ] 已确定各层默认生命周期。
- [ ] 已确定配置对象注册方式。
- [ ] 已确定多实现接口的选择方式。
- [ ] 已确定测试环境如何替换依赖。
- [ ] 已避免在业务代码中直接解析容器。
- [ ] 如果使用 Autofac，已阅读并采用 [AUTOFAC_GUIDE.md](AUTOFAC_GUIDE.md)。

## 八、参数校验

- [ ] 已确定 Controller 只做轻量校验。
- [ ] 已确定复杂业务校验放在 Service 或专门校验组件中。
- [ ] 已确定 DTO 校验方式。
- [ ] 已确定参数校验失败的错误码和响应格式。
- [ ] 已覆盖必填、空白字符串、数字越界、日期范围、分页参数、枚举非法值。

## 九、异常和日志

- [ ] 已接入全局异常处理。
- [ ] 已确定不可预期异常的统一响应格式。
- [ ] 已确定日志框架和日志级别。
- [ ] 已确定 traceId / requestId 贯穿方式。
- [ ] 已确定哪些业务动作必须记录日志。
- [ ] 已确定外部系统调用日志规则。
- [ ] 已明确敏感信息不进入日志。

## 十、数据访问

- [ ] 已确定使用 Repository、DbContext 或其他数据访问模式。
- [ ] 已确定泛型 Repository、专用 Repository、Query Service 的边界。
- [ ] 已确定是否允许 Repository 暴露 IQueryable。
- [ ] 已确定 SaveChanges 由谁负责。
- [ ] 已确定只读查询是否默认无跟踪。
- [ ] 已确定实体映射规则。
- [ ] 已确定分页查询规则。
- [ ] 已确定排序和过滤规则。
- [ ] 已确定软删除、审计字段、并发控制是否需要。
- [ ] 已确定如何避免 N+1 查询。
- [ ] 已确定数据库异常如何转换为业务结果或系统异常。
- [ ] 已阅读并采用 [REPOSITORY_GUIDE.md](REPOSITORY_GUIDE.md)。

## 十一、事务和一致性

- [ ] 已确定 Service 方法是否作为默认事务边界。
- [ ] 已确定跨多个写操作时的事务机制。
- [ ] 已确定外部系统调用和数据库写入混合时的补偿策略。
- [ ] 已确定哪些写接口需要幂等。
- [ ] 已确定幂等键、业务唯一约束或状态机防重策略。
- [ ] 已确定重复提交时返回稳定结果。

## 十二、缓存

- [ ] 已确定是否需要缓存。
- [ ] 已确定缓存 key 规则。
- [ ] 已确定缓存过期策略。
- [ ] 已确定写操作后的缓存更新或失效策略。
- [ ] 已确定缓存穿透、击穿、雪崩是否需要处理。
- [ ] 已避免用缓存掩盖数据一致性问题。

## 十三、外部集成

- [ ] 已确定第三方系统访问是否封装在外部集成层。
- [ ] 已确定超时策略。
- [ ] 已确定重试策略。
- [ ] 已确定失败日志和 traceId 记录方式。
- [ ] 已确定限流、熔断或降级是否需要。
- [ ] 已确定外部调用的幂等策略。
- [ ] 已避免把第三方 SDK 类型直接暴露给 API 层。

## 十四、异步和性能

- [ ] IO 操作优先使用异步。
- [ ] 已避免 `.Result` 和 `.Wait()`。
- [ ] 已确定列表接口最大分页大小。
- [ ] 已避免接口返回不必要的大对象。
- [ ] 已检查关键列表接口是否存在 N+1 查询。
- [ ] 已确定慢查询、外部慢调用的监控方式。

## 十五、接口文档

- [ ] 已接入接口文档生成工具。
- [ ] 已补充请求参数说明。
- [ ] 已补充响应示例。
- [ ] 已补充错误码说明。
- [ ] 已补充分页规则。
- [ ] 已补充幂等规则。
- [ ] 已补充重要字段说明。
- [ ] 文档示例与统一结果对象一致。

## 十六、测试

- [ ] 已准备 Controller 轻量集成测试。
- [ ] 已准备 Service 业务测试。
- [ ] 已准备数据访问关键查询测试。
- [ ] 已准备外部依赖 mock / stub / 测试替身。
- [ ] 已准备容器构建测试。
- [ ] 已准备统一结果对象响应测试。
- [ ] 已覆盖成功路径、业务失败、参数非法、分页默认值、边界值、重复提交。

## 十七、上线前检查

- [ ] 关键配置已在目标环境存在。
- [ ] 日志能输出 traceId / requestId。
- [ ] 全局异常处理生效。
- [ ] 接口文档可访问。
- [ ] 健康检查可用。
- [ ] 数据库迁移或初始化步骤明确。
- [ ] 外部系统配置已验证。
- [ ] 关键写接口幂等策略已验证。
- [ ] 基础性能风险已检查。
- [ ] 回滚方案已明确。

## 十八、最小可用标准

新项目至少应具备：

- [ ] 清晰分层。
- [ ] 统一 API 结果对象。
- [ ] 集中依赖注入注册。
- [ ] 全局异常处理。
- [ ] traceId / requestId。
- [ ] 配置绑定和关键配置校验。
- [ ] 接口文档。
- [ ] 最小测试集。
- [ ] 一个典型查询接口。
- [ ] 一个典型写接口。
- [ ] 一个业务失败示例。
